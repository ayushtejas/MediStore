Section 2 — Auth Flow

NextAuth.js + FastAPI JWT
Login (Staff/Admin):
1. POST /api/auth/signin → NextAuth Credentials provider
2. NextAuth calls FastAPI POST /auth/login with email+password
3. FastAPI verifies, returns access_token (JWT) + role
4. NextAuth stores token in encrypted session cookie
5. Subsequent requests: BFF proxy reads session, attaches Bearer token → FastAPI

Login (Customer):
Same flow via NextAuth Credentials — role = "customer"

Roles:  admin  staff  customer
middleware.ts guards:
 /admin/* → requires admin
 /pos/* → requires admin | staff
 /(shop)/orders → requires customer

Section 5 — Cross-cutting concerns

  • ORM: SQLAlchemy 2.x (async) + Alembic for migrations
  • Validation: Pydantic v2 schemas on all request/response bodies
  • Inventory sync: All stock deductions run inside a DB transaction with row-level locking (SELECT FOR UPDATE) — prevents overselling
  • FIFO: GET /orders/{id}/items auto-selects oldest non-expired batch first
  • Background jobs: FastAPI APScheduler for daily expiry + low-stock alerts
  • Docker Compose: postgres:16, redis:7, fastapi service — single docker compose up
  • Testing: pytest + httpx for FastAPI; Jest + React Testing Library for Next.js components
