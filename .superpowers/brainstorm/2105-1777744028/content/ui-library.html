UI Component Library

Which foundation should we build the Next.js frontend on?

A

shadcn/ui + Tailwind CSS

Copy-paste components (not a package dependency). Fully customisable, Tailwind-based, excellent DX. Industry standard for modern Next.js apps. Radix UI primitives underneath for accessibility.

Pros

  • Own the code — no library lock-in
  • Beautiful out of the box
  • Best Next.js App Router support
  • Huge ecosystem and community

Cons

  • Need to compose data tables, forms yourself
B

Ant Design (antd)

Enterprise-grade component library with a comprehensive set of pre-built components — data tables, forms, date pickers, layout. Popular for admin panels and dashboards in India/Asia.

Pros

  • Rich data table + form components
  • Admin-ready out of the box
  • Less layout work for POS/dashboard

Cons

  • Large bundle size
  • Less flexible to customise
  • Older design aesthetic
C

Chakra UI

Accessible, composable React component library. Simpler theming model than shadcn, easier learning curve. Good for teams that want consistency without much setup.

Pros

  • Easy theming + dark mode
  • Good accessibility defaults
  • Gentle learning curve

Cons

  • Smaller ecosystem than shadcn
  • App Router support requires extra config

Recommendation

A — shadcn/ui + Tailwind is the best fit for this project. The medical UI folder name suggests a polished look is a priority, and shadcn gives you full control over every pixel while shipping fast. For the data-heavy admin panel, we'll add TanStack Table on top for the POS and inventory tables.
