Key decisions captured here

  • FastAPI uses domain modules (inventory, orders, auth, users, payments) — each with its own models/schemas/router/service. Clean, portable when extracted to its own repo.
  • Next.js route groups (shop), admin, pos isolate concerns without affecting the URL structure.
  • middleware.ts enforces auth at the edge — unauthenticated users are redirected before any page renders.
  • The api/[...proxy] BFF route forwards requests from React Query to FastAPI, attaching the JWT server-side so FastAPI never sees browser requests directly.
