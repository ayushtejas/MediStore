# Windows Desktop Install Guide

MedStore Offline is packaged as a Windows desktop app. The client should not install
Python, Node.js, SQLite, Postgres, or run any commands after installation.

The final client experience is:

1. Receive `MedStore Offline Setup.exe`.
2. Double-click the installer.
3. Open `MedStore Offline` from Start Menu or Desktop.
4. Login and start using Admin/POS offline.

## What Runs After Install

The installed app starts everything it needs by itself:

| Part | Runs As | Purpose |
| --- | --- | --- |
| Electron | Windows desktop app | Opens the application window |
| Next.js standalone server | Bundled local process | Serves the Admin/POS UI |
| FastAPI backend | Bundled `medstore-api.exe` | Handles auth, inventory, billing and reports |
| SQLite | Local database file | Stores medicines, stock, suppliers and bills |

There is no cloud dependency and no external database dependency.

## Client Installation Steps

Give the client only this file:

```text
MedStore Offline Setup.exe
```

On the client's Windows computer:

1. Copy `MedStore Offline Setup.exe` to the computer.
2. Double-click the installer.
3. If Windows SmartScreen appears, click `More info`, then `Run anyway`.
4. Wait for installation to finish.
5. Open `MedStore Offline` from the Start Menu.
6. Login with an Admin or Counter Staff account.

If the installer is code-signed with a trusted certificate, the SmartScreen warning should
be reduced or avoided.

## Default Logins

| Role | Email | Password |
| --- | --- | --- |
| Admin | `admin@medstore.dev` | `Admin@123` |
| Counter staff | `staff@medstore.dev` | `Staff@123` |

Change these before handing the app to a real store.

## Where Client Data Is Stored

The app stores the SQLite database in the Windows user data folder.

Typical path:

```text
%APPDATA%\MedStore Offline\medical_store.db
```

This file contains the client's local business data: medicines, batches, stock,
suppliers, bills and reports.

For backup:

1. Close the app.
2. Copy `medical_store.db` to a safe location.
3. To restore, close the app and replace the same file.

## Build Machine Requirements

These are required only on our build machine, not on the client's computer.

| Tool | Used For |
| --- | --- |
| Windows 10/11 | Building the Windows installer |
| Node.js 20+ | Building Next/Electron |
| Python 3.12 | Building the FastAPI backend exe |
| PowerShell | Running build commands |

## Build The Backend EXE

Run from PowerShell in the project root:

```powershell
cd modern_medical_backend
py -m venv .venv
.\.venv\Scripts\python -m pip install --upgrade pip
.\.venv\Scripts\pip install -r requirements.txt pyinstaller
.\.venv\Scripts\pyinstaller --name medstore-api --onefile --collect-all app --collect-all alembic desktop_entry.py
New-Item -ItemType Directory -Force ..\desktop-dist\backend
Copy-Item .\dist\medstore-api.exe ..\desktop-dist\backend\medstore-api.exe -Force
cd ..
```

Expected file:

```text
desktop-dist\backend\medstore-api.exe
```

## Build The Windows Installer

Run from PowerShell in the project root:

```powershell
npm install
npm run desktop:make
```

Expected output folder:

```text
out\make\squirrel.windows\x64\
```

Share this installer with the client:

```text
out\make\squirrel.windows\x64\MedStore Offline Setup.exe
```

Do not ask the client to run `npm install`, `pyinstaller`, `uvicorn`, or any terminal command.

## Local Development

For browser-based development:

```bash
npm run dev
```

This starts:

| Service | URL |
| --- | --- |
| Next frontend | `http://127.0.0.1:3000` |
| FastAPI backend | `http://127.0.0.1:8000` |

For Electron-window development:

```bash
npm run desktop:dev
```

This starts:

| Service | URL |
| --- | --- |
| Next frontend | `http://127.0.0.1:3210` |
| FastAPI backend | `http://127.0.0.1:8000` |

## Notes

- `desktop-dist/frontend` is generated from the Next standalone build.
- `desktop-dist/backend/medstore-api.exe` is generated by PyInstaller.
- The installed app calls its own local API on `127.0.0.1`.
- The local API is intended for this desktop app only, not for network access.
- For production distribution, sign the installer so Windows trusts it.
